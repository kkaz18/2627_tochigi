# 栃木SC2026/27 選手一覧 — 公開・更新ガイド（GitHub Pages）

このフォルダの2ファイルを GitHub Pages に置くと、無料でWebサイトとして公開できます。

- `index.html` … アプリ本体（デザイン・機能）。基本そのまま。
- `roster-data.js` … 選手データ（公開される名簿）。更新時はこれを差し替えます。

---

## 1. 最初の公開（一度だけ）

1. GitHub アカウントを作成（無料）: https://github.com/
2. 右上「＋」→「New repository」でリポジトリを作成。
   - Repository name: 例 `tochigi-roster`
   - 「Public」を選択 →「Create repository」
3. リポジトリ画面で「Add file」→「Upload files」を開き、
   `index.html` と `roster-data.js` をドラッグして「Commit changes」。
4. 上部メニュー「Settings」→ 左メニュー「Pages」を開く。
5. 「Build and deployment」の Source を **Deploy from a branch**、
   Branch を **main** / **/(root)** にして「Save」。
6. 数分後、ページ上部に公開URLが表示されます。
   例: `https://（ユーザー名）.github.io/tochigi-roster/`
   → このURLが公開サイトです。スマホでもPCでも開けます。

> 独自ドメイン（例: tochigisc-fan.com など）も Settings → Pages → Custom domain から後で設定できます。

---

## 2. 選手情報を更新する（継続運用）

1. 公開URLを開く。
2. 画面左上の**ロゴを5秒間長押し** → パスワード **`2026KKAZ`** を入力して管理モードに入る。
3. 「＋ 選手を追加」「各カードの×で削除」「プロフィール右上の編集」で内容を編集。
   - 背番号、写真（最大4枚）、移籍区分、各プロフィール項目などを変更できます。
4. ヘッダーの「**⤓ データを書き出す**」を押す（編集があると赤い印が付きます）。
   → `roster-data.js` がダウンロードされます。
5. GitHub のリポジトリで `roster-data.js` を開き、「Add file」→「Upload files」で
   ダウンロードした `roster-data.js` を**同じ名前のままアップロード（上書き）**→「Commit changes」。
6. 数分でサイトに反映されます。

> 編集はまず「自分の端末内」に保存され、5〜6の手順で公開に反映される仕組みです。
> 端末内の編集を取り消して公開中の状態に戻したいときは、管理モードの「編集を破棄」を押します。

---

## 3. 注意点

- **パスワードは簡易ロックです。** ページのソースに含まれるため、技術的には閲覧者から見える可能性があります。本格的な権限管理ではなく「うっかり編集の防止」程度とお考えください。サイト自体は誰でも閲覧できます。
- **写真はデータに埋め込まれます。** 多数・大容量の写真を入れると `roster-data.js` が大きくなり、読み込みが遅くなります。1選手あたりの枚数を絞る、画像は自動圧縮に任せる、などで軽量化できます（アップロード時に自動縮小されます）。
- **公式サイトの選手写真は自動取得していません。** 各選手の写真はご自身で用意・アップロードしてください。
- `index.html` のデザインや機能を変えたい場合は、その都度 `index.html` を新しいものに差し替えてください（データの `roster-data.js` はそのまま使えます）。

---

## 4. うまく表示されないとき

- ページが真っ白 → `index.html` と `roster-data.js` が**同じ階層（同じフォルダ）**にあるか確認。
- 更新が反映されない → ブラウザの再読み込み（キャッシュ削除）。GitHub Pages は反映に数分かかることがあります。
- 写真が出ない → 画像URLの綴り、またはアップロードのやり直しを確認。
